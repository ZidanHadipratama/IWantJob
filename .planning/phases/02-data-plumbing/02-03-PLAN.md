  <action>
    **1. Create `extension/src/lib/storage.ts`:**
    - Type definitions for storage keys:
      ```typescript
      interface AIConfig { provider: string; api_key: string; model: string }
      interface DBConfig { supabase_url: string; supabase_key: string }
      interface UserProfileConfig { name: string; email: string; linkedin_url: string; github_url: string; work_authorization: string }
      interface StorageSchema {
        ai_config: AIConfig;
        db_config: DBConfig;
        backend_url: string;
        user_profile: UserProfileConfig;
        user_id: string;
        base_resume_text: string;
      }
      ```
    - `getStorage<K extends keyof StorageSchema>(key: K): Promise<StorageSchema[K] | null>` — wraps chrome.storage.local.get
    - `setStorage<K extends keyof StorageSchema>(key: K, value: StorageSchema[K]): Promise<void>` — wraps chrome.storage.local.set
    - `getOrCreateUserId(): Promise<string>` — gets user_id from storage, or generates UUID via crypto.randomUUID() and stores it
    - Export all types and functions

    **2. Create `extension/src/lib/api.ts`:**
    - `createApiClient()` async function that:
      - Reads backend_url, db_config, user_id from storage
      - Returns an object with methods:
        - `testConnection(): Promise<{connected: boolean, message: string}>` — GET /test-connection with Supabase + user headers
        - `uploadResume(file: File): Promise<{resume_text: string, message: string}>` — POST /upload-resume with FormData
        - `saveResumeText(text: string): Promise<{resume_text: string, message: string}>` — POST /save-resume-text with JSON body
      - All methods inject headers: X-Supabase-Url, X-Supabase-Key, X-User-Id
      - Handle network errors gracefully — return descriptive error messages, don't throw
