## Fix 1: Supabase Connection — use service_role key, not anon key

**Root cause:** RLS policies use `auth.uid()` which requires Supabase Auth JWT. We use a custom UUID (X-User-Id header), not Supabase Auth. With the anon key, `auth.uid()` returns null and RLS blocks all queries.

**Fix:**
- Backend: No code change needed — `create_client(url, key)` works the same with service_role key
- Extension `SupabaseConfigCard.tsx` (line 102-103): Change label from "Supabase Anon Key" to "Supabase Service Role Key"
- Add helper text: "Find this in Supabase Dashboard → Settings → API → service_role key (secret)"
- Extension `lib/storage.ts`: Rename `supabase_key` field in DBConfig type to `supabase_service_key` (or keep as-is, just update labels)
- Update `extension/src/lib/api.ts` header name if field renamed

**Files to modify:**
- extension/src/components/options/SupabaseConfigCard.tsx (labels + help text)
- Optionally: extension/src/lib/storage.ts (type rename for clarity)

## Fix 2: PDF Extraction — replace pdfplumber with Marker

**Root cause:** pdfplumber only extracts embedded text layers. Image-based/scanned/Canva resumes fail.

**User chose:** marker-pdf (ML-based, handles all PDF types)

**Fix:**
- `backend/requirements.txt`: Replace `pdfplumber` with `marker-pdf`
- `backend/Dockerfile`: Add PyTorch install (pip install torch --index-url https://download.pytorch.org/whl/cpu for CPU-only, keeps image smaller)
- `backend/app/routers/resume.py`: Replace pdfplumber extraction with Marker:
  ```python
  from marker.converters.pdf import PdfConverter
  from marker.models import create_model_dict

  converter = PdfConverter(artifact_dict=create_model_dict())
  rendered = converter(temp_file_path)
  extracted_text = rendered.markdown  # or .text
  ```
- Note: Marker needs a file path, not BytesIO — write to tempfile first
- Update error messages accordingly
- Update tests in test_endpoints.py that mock pdfplumber

**Files to modify:**
- backend/requirements.txt
- backend/Dockerfile
- backend/app/routers/resume.py
- backend/tests/test_endpoints.py
