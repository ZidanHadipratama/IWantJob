    1. Create `backend/app/dependencies.py`:
       - `get_supabase_client(request: Request) -> Client`: Extract X-Supabase-Url and X-Supabase-Key from request headers. If either missing, raise HTTPException(400, detail="Missing X-Supabase-Url header" or "Missing X-Supabase-Key header"). Call `supabase.create_client(url, key)` and return the sync Client. Use `from supabase import create_client, Client`.
       - `get_user_id(request: Request) -> str`: Extract X-User-Id header. If missing, raise HTTPException(400, detail="Missing X-User-Id header"). Return the string value.

    2. Add new schemas to `backend/app/models/schemas.py`:
       - `SaveQARequest(BaseModel)`: job_id: UUID, qa_pairs: list[QAPair]
       - `QAPair(BaseModel)`: field_id: str, question: str, answer: str, field_type: str = "text", edited_by_user: bool = False
       - `JobResponse(BaseModel)`: id: UUID, company: str, title: str, url: Optional[str], job_description: Optional[str], status: str, applied_at: Optional[str], notes: Optional[str], created_at: str, qa_pairs: Optional[list[dict]] = None, resumes: Optional[list[dict]] = None, chat_messages: Optional[list[dict]] = None
       - `LogJobResponse(BaseModel)`: id: UUID, company: str, title: str, status: str, created_at: str
       - `TestConnectionResponse(BaseModel)`: connected: bool, message: str

    3. Create `backend/app/services/db_service.py`:
       - Class `DBService` with `__init__(self, client: Client, user_id: str)` storing both.
       - Methods (all sync, supabase-py v2 sync API):
         - `upsert_user(data: dict) -> dict`: upsert into users with id=user_id
         - `get_user() -> dict | None`: select from users where id=user_id, return single or None
         - `create_job(data: dict) -> dict`: insert into jobs with user_id, return inserted row
         - `update_job(job_id: str, data: dict) -> dict`: update jobs where id=job_id AND user_id, return updated row
         - `get_job(job_id: str) -> dict | None`: select from jobs where id=job_id AND user_id
         - `get_jobs_for_user() -> list[dict]`: select from jobs where user_id, ordered by created_at desc
         - `upsert_qa_pairs(job_id: str, qa_pairs: list[dict]) -> list[dict]`: for each pair, set job_id, user_id, upsert with on_conflict="job_id,field_id"
         - `get_qa_pairs_for_job(job_id: str) -> list[dict]`: select from form_qa_pairs where job_id AND user_id
         - `get_resumes_for_job(job_id: str) -> list[dict]`: select from resumes where job_id AND user_id
         - `get_chat_messages_for_job(job_id: str) -> list[dict]`: select from chat_messages where job_id AND user_id
         - `save_base_resume(resume_text: str) -> dict`: upsert into resumes with user_id, is_base=True, job_id=None
         - `get_base_resume() -> dict | None`: select from resumes where user_id AND is_base=True

       All supabase calls use the pattern: `self.client.table("tablename").select/insert/upsert/update(...)...execute()` and return `.data`.
